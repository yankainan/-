November 8, 2023

团队名称

嘛也没学会

项目描述

团队的目标是开发一款名为 “TaskEase” 的任务管理应用程序，旨在提供简单而强大的任务管理功能，以帮助用户更高效地管理他们的日常任务和工作流程。我们希望了解潜在用户的需求、工作环境以及他们在任务管理方面的体验，以确保 “TaskEase” 能够满足他们的需求并为他们提供有价值的功能。

团队分工

颜恺楠：负责调查分析部分，是这一部分的主要原始撰写者，并进行初步的数据整理和分析。最后，参与了最终的审阅和改进。

曾俊豪：负责亲和图表的制作，并将原始数据转化为数字版本，是亲和图表的主要原始撰写者。

赵书健：负责调查问卷设计和数据收集，参与了调查的执行和数据整理。此外，也参与了亲和图表的数字编辑，以确保数据的准确性

师语：在整个过程中负责团队的协调和组织，确保每个部分按计划进行，还协助审阅和改进了整个报告，以确保一致性和完整性。

调查分析

在我们的调查中，我们收集了来自各行各业的各种回答，以了解他们的任务管理需求和体验。通过合适的可视化格式，我们能够清晰地呈现了一些重要的发现。最显著的发现之一是关于任务管理挑战的。根据我们的数据，约70%的受访者提到了任务分配和时间管理方面的挑战。这些问题被呈现为柱状图，显示了这一趋势。

在我们的调查样本中，受访者的年龄分布相对均匀，大多数人处于25到35岁之间，表明任务管理工具的需求在各年龄段都普遍存在。通过描述性统计数据，我们能够清晰地了解受访者的年龄分布情况。下面是统计可视化图

此外，我们还了解到受访者在工作之外有各种兴趣爱好或活动。这些包括锻炼、阅读、社交活动等。这些信息对于我们了解受访者的日常生活和需求非常有帮助。通过了解受访者的兴趣爱好，我们可以更好地了解他们的生活方式和时间分配情况。这有助于我们在设计 “TaskEase” 应用时考虑到用户的日常活动，并提供更贴近他们需求的功能。下面是他们的可视化图

通过调查数据分析，我们学到了任务管理在受访者中非常普遍，但同时也存在一些共同的挑战。任务分配和时间管理问题是普遍存在的，但有一些意外的发现。有一部分受访者表示他们愿意更多地使用任务管理工具，前提是这些工具提供更好的团队协作功能。这一发现表明，任务管理工具的未来发展可能需要更多的团队协作功能。此外，我们还观察到了不同行业和年龄组之间的一些差异。一些行业更注重时间管理，而另一些则更关注任务分配。这些差异将有助于我们更好地满足不同用户群体的需求。这些发现对 “TaskEase” 应用的设计非常有指导意义。我们可以优化工具以更好地满足不同用户的需求，从而提供更高效的任务管理体验。

亲和力图

根据亲和力图的总结

  1. 图表包含重要主题？

    我们的亲和图表包含了关于任务管理的重要主题。这些主题涵盖了任务分配、时间管理、团队协作等关键领域。通过图表，我们能够清晰地看到这些主题的关联性，以及它们在任务管理中的重要性。

    任务分配是一个主题，它包括了与任务分配相关的所有子主题。在图表中，任务分配主题下有子主题，如任务分配混乱、任务分配工具等。这种结构有助于我们更好地理解各个主题之间的关系。

  2. 最终图表具有主题和子主题的结构/关系。

    亲和图表的结构非常清晰，主题和子主题之间的关系得以明确呈现。这有助于我们更深入地理解任务管理领域的各个方面。

    例如，主题”时间管理”下有多个子主题，如”时间管理工具”、”时间浪费”等。这些子主题通过连线清晰地展示了它们之间的关联性。这有助于我们更有针对性地分析用户需求，以便在 “TaskEase” 应用中提供更多有价值的功能。

  3. 我们学到了什么？

    通过亲和图表分析，我们进一步加深了对任务管理主题的理解。我们了解到不同主题之间的关联性，以及受访者对任务管理工具的期望。一些意外的发现包括受访者对更好的任务分配和信息集中的渴望。这将有助于我们在 “TaskEase” 应用中提供更有价值的功能。

云计算技术 课程实验报告

学号：202000130203 姓名：颜恺楠 班级：数据
实验题目：利用云平台搭建个人博客
实验学时：2 实验日期： 2023.3.14
实验目的：熟悉个人博客系统的搭建。 具体包括： 参考方案：注册Github账号，搭建Hexo环境并实现个人博客搭建，撰写实验报告。
硬件环境： 联网的计算机一台
软件环境： Windows or Linux
实验步骤与内容： 在这个实验中遇到了很多坑，我将一步步解释我是如何做这个实验并附上一些解决网址链接和出问题时的界面。 1、首先要注册一个github账号，这一步不再多说，直接根据github网站上的注册步骤走就行，注册结束后需要创建一个新的仓库，进入自己的主页后按照下面顺序操作 则可以创建一个新的仓库，注意这个仓库要命名为“用户名.github.io”。这是之后要登陆博客界面时候的网址，一定要按照规范来。接下来仓库就构建完毕了。我的github主页网址是https://github.com/yankainan/，创建成功后页面如下图。 2、安装Git和node.js两个软件，其中node.js下载地址是http://nodejs.cn，Git下载地址是https://git-scm.com/，下载后Git就会出现这五个小软件 可以使用命令控制台来查看node.js和Git的版本，以此验证安装是否成功 我的验证如下图，说明是成功的 3、在本地文件夹中中创建一个用来存放Github本地仓库文件的目录，我创建的是”D:\云计算\blog” 4、选中创建的目录(文件夹)右键选择使用Gti Bash Here打开Git命令窗口。 输入命令npm install -g cnpm –registry=https://registry.npm.taobao.org 但是出现了这个错 ，我将报错信息放到网上搜索，参考了以下这篇博客，https://blog.csdn.net/qq_44075310/article/details/128255366，首先按照他第二点说的更改用户的权限，但是仍然不起作用。之后我进入C盘，在用户名目录下的.npmrc文件删除后，就可以成功安装，结果如下图。 之后输入cnpm install -g hexo-cli命令进行hexo的安装，在经过前面bug的解决后，这一步也可以成功，结果如下图。 5、接下来应该对当前的文件夹进行hexo初始化，于是输入了hexo init命令，但是遇到了下面这个问题 我又用hexo v查看hexo版本验证时候安装成功，但是也遇到了相同的问题： 于是我参考了这篇博客https://blog.csdn.net/qq_42893625/article/details/100852221，在命令前加上了npx，在结尾也加上了当前的文件夹的名称“blog”，就成功了，结果如下图： 且查看版本号为 在之后的涉及hexo的命令中我也使用的npx前缀进行操作。 这时我们创建的目录(文件夹)下已经多出许多文件(这里不多介绍这些文件及目录)。 6、使用npm install hexo-server –save命令安装hexo的服务器，这样可以使用npx hexo server启动服务器，之后使用http://localhost:4000/直接进行博客地址的访问，安装结果如下图 之后启动服务器的结果为： 当要结束时，可以使用ctrl+c来关闭服务器。本地进入博客的界面忘记截图了，到时候直接看最后的结果吧。 7、设置ssh。 输入cd ~/.ssh进入.ssh文件，之后使用ssh-keygen -t rsa -C ‘注册时的邮箱地址’，此时查看C:\用户\用户名目录下的.ssh文件夹的id_ras.pub文件中的ssh如下图： 复制上图的ssh密钥，在已登陆的Github主页点击头像，之后点击Settings设置，并按下图顺序点击，创建新的ssh密钥，随便取一个名字，结果如下： 这里还需要在本地进行验证一次，输入命令：ssh -T git@github.com 并且输入yes之后，行末尾会显示你的用户名，如下 接着在本地绑定与Github的用户名和邮箱 输入命令：git config –global user.name “注册时用户名” 输入命令：git config –global user.email “注册时邮箱” 8、接下来用记事本打开并修改本地仓库目录下的_config.yml文件，最后一段更改为如下 # Deployment ## 结论分析与体会： 通过这个实验，我对于使用github和hexo结合来部署自己的博客网站有了更深的认识，可以熟练操作、搭建、添加、编辑博客。在实验的最后我也成功创建且编辑了一篇博客，并上传到github，虽然在实验中我遇到了诸如npm无法操作成功使用，直接使用hexo无法成功的罕见问题，但是我仍通过上网搜索、不断尝试来解决了这些问题。例如经过原因查询，在hexo前面加上npx就能成功的原因是：npx 想要解决的主要问题，就是调用项目内部安装的模块，所以可以理解为在命令行下调用，可以让项目内部安装的模块用起来更方便，npx运行的时候，会到node_modules/.bin路径和环境变量$PATH里面，检查命令是否存在，所以系统命令也可以调用，即上面的直接hexo init命令安装不成功的时候加上npx的话也许就可以成功了。并且经过实验，我认为使用github和hexo来搭建博客可以让博客的文件保持在本地，无需网路就能打开，极大的方便了用户。
