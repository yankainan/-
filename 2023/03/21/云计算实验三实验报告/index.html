云计算技术 课程实验报告

学号：202000130203 姓名：颜恺楠 班级：数据
实验题目：利用云平台搭建个人博客
实验学时：2 实验日期： 2023.3.14
实验目的：熟悉个人博客系统的搭建。 具体包括： 参考方案：注册Github账号，搭建Hexo环境并实现个人博客搭建，撰写实验报告。
硬件环境： 联网的计算机一台
软件环境： Windows or Linux
实验步骤与内容： 在这个实验中遇到了很多坑，我将一步步解释我是如何做这个实验并附上一些解决网址链接和出问题时的界面。 1、首先要注册一个github账号，这一步不再多说，直接根据github网站上的注册步骤走就行，注册结束后需要创建一个新的仓库，进入自己的主页后按照下面顺序操作 则可以创建一个新的仓库，注意这个仓库要命名为“用户名.github.io”。这是之后要登陆博客界面时候的网址，一定要按照规范来。接下来仓库就构建完毕了。我的github主页网址是https://github.com/yankainan/，创建成功后页面如下图。 2、安装Git和node.js两个软件，其中node.js下载地址是http://nodejs.cn，Git下载地址是https://git-scm.com/，下载后Git就会出现这五个小软件 可以使用命令控制台来查看node.js和Git的版本，以此验证安装是否成功 我的验证如下图，说明是成功的 3、在本地文件夹中中创建一个用来存放Github本地仓库文件的目录，我创建的是”D:\云计算\blog” 4、选中创建的目录(文件夹)右键选择使用Gti Bash Here打开Git命令窗口。 输入命令npm install -g cnpm –registry=https://registry.npm.taobao.org 但是出现了这个错 ，我将报错信息放到网上搜索，参考了以下这篇博客，https://blog.csdn.net/qq_44075310/article/details/128255366，首先按照他第二点说的更改用户的权限，但是仍然不起作用。之后我进入C盘，在用户名目录下的.npmrc文件删除后，就可以成功安装，结果如下图。 之后输入cnpm install -g hexo-cli命令进行hexo的安装，在经过前面bug的解决后，这一步也可以成功，结果如下图。 5、接下来应该对当前的文件夹进行hexo初始化，于是输入了hexo init命令，但是遇到了下面这个问题 我又用hexo v查看hexo版本验证时候安装成功，但是也遇到了相同的问题： 于是我参考了这篇博客https://blog.csdn.net/qq_42893625/article/details/100852221，在命令前加上了npx，在结尾也加上了当前的文件夹的名称“blog”，就成功了，结果如下图： 且查看版本号为 在之后的涉及hexo的命令中我也使用的npx前缀进行操作。 这时我们创建的目录(文件夹)下已经多出许多文件(这里不多介绍这些文件及目录)。 6、使用npm install hexo-server –save命令安装hexo的服务器，这样可以使用npx hexo server启动服务器，之后使用http://localhost:4000/直接进行博客地址的访问，安装结果如下图 之后启动服务器的结果为： 当要结束时，可以使用ctrl+c来关闭服务器。本地进入博客的界面忘记截图了，到时候直接看最后的结果吧。 7、设置ssh。 输入cd ~/.ssh进入.ssh文件，之后使用ssh-keygen -t rsa -C ‘注册时的邮箱地址’，此时查看C:\用户\用户名目录下的.ssh文件夹的id_ras.pub文件中的ssh如下图： 复制上图的ssh密钥，在已登陆的Github主页点击头像，之后点击Settings设置，并按下图顺序点击，创建新的ssh密钥，随便取一个名字，结果如下： 这里还需要在本地进行验证一次，输入命令：ssh -T git@github.com 并且输入yes之后，行末尾会显示你的用户名，如下 接着在本地绑定与Github的用户名和邮箱 输入命令：git config –global user.name “注册时用户名” 输入命令：git config –global user.email “注册时邮箱” 8、接下来用记事本打开并修改本地仓库目录下的_config.yml文件，最后一段更改为如下 # Deployment ## Docs: https://hexo.io/docs/one-command-deployment deploy: type: git repository: git@github.com:yankainan/yankainan.github.io.git branch: main 注意缩进和用户名更改，以及冒号之后有一个空格。 现在还需要安装一个工具cnpm install hexo-deployer-git，用来更新博客在github中。 9、可以新建一篇测试文章(不新建的话也会有一篇自带的Hello World文章)，输入命令：hexo new “文章名称”，如下图 新建文章之后需要生成一遍文件，输入命令：npx hexo g。 新建文章后可在本地先预览一遍，输入命令：npx hexo s浏览器输入：http://localhost:4000(ctrl + c 关闭) 最后直接部署到Github，使用命令：npx hexo d 此时已经成功上传github，查看仓库，也多了一些文件夹 10、现在发现博客里没内容，需要进行内容的编辑。步骤是进入该地址”D:\云计算\blog\blog\source_posts"，之后就可以在这里面选择要更改的博客进行内容编辑，现在还只有两篇博客。 之后使用npx hexo g和npx hexo d来重新生成文件并上传至github，进入网页访问如下图，发现已经成功更新。 可以使用https://yankainan.github.io/来访问我的博客 实验****3.2 对于实验3.2，目标是要创建一篇新的博客，并上传上面的实验报告。如何创建新的博客和如何进行更改在上文已经有阐述。完成上面的步骤后，使用http://yankainan.github.io/来进行访问，我还进行了主题的更改，步骤如下： 在hexo官网中下载目标主题的github库，放入本地的根目录中，更改_config.yaml文件中的theme选项为放入本地根目录的文件名即可。之后我又对右边的github链接和微博链接做了一个更改，可以相应的点进去查看。 结果如下（仅截取部分，具体可以登陆网站看）：
结论分析与体会： 通过这个实验，我对于使用github和hexo结合来部署自己的博客网站有了更深的认识，可以熟练操作、搭建、添加、编辑博客。在实验的最后我也成功创建且编辑了一篇博客，并上传到github，虽然在实验中我遇到了诸如npm无法操作成功使用，直接使用hexo无法成功的罕见问题，但是我仍通过上网搜索、不断尝试来解决了这些问题。例如经过原因查询，在hexo前面加上npx就能成功的原因是：npx 想要解决的主要问题，就是调用项目内部安装的模块，所以可以理解为在命令行下调用，可以让项目内部安装的模块用起来更方便，npx运行的时候，会到node_modules/.bin路径和环境变量$PATH里面，检查命令是否存在，所以系统命令也可以调用，即上面的直接hexo init命令安装不成功的时候加上npx的话也许就可以成功了。并且经过实验，我认为使用github和hexo来搭建博客可以让博客的文件保持在本地，无需网路就能打开，极大的方便了用户。
